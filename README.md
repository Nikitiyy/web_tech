# Каталог товаров музыкального магазина
- Это приложение представляет собой каталог товаров музыкального магазина. В нём содержится весь ассортимент товара, доступный для покупки либо для предзаказа. 
- Целевая аудитория: покупатели, заинтересованные в приобритении музыкальных инструментов либо музыкального оборудования.
- Приложение позволяет просматривать товары, добавлять их в корзину, бронировать, а также пользоваться помощью консультантов.
# Структура приложения
- Аутентификация (`/`)
- Главная страница, ознакомительная (`/main_menu`)
- Меню для гостей (`/guest_menu`)
- Страница профиля (`/profile`)
- Категории товаров (`/categories`)
- Страница с товарами (`/products`)
- Страница товара (`/product/{id}`)
- Корзина (`/cart`)
- Восстановление пароля (`/recovery`, `/reset-password`)
- Админ панель (`/main_admin`)
- Управление товарами (`/admin-products`)
- Управление категориями (`/admin-categories`)
- Управление администраторами (`/admin-admins`)
- Бронирования (`/admin-reservations`)
- Изменение/создание товара (`/admin-edit-products`, `/admin-add-products`)
# Frontend 
Для реализации frontend был выбран чистый JS.
Данный выбор обусловлен следующими причинами:
- Уменьшение сложности проекта за счёт отсутствия дополнительных зависимостей
- Полный контроль над кодом и архитектурой приложения
- Высокая производительность за счёт отсутствия лишних абстракций
- Целесообразность использования при небольшом или среднем масштабе проекта.
Для организации навигации в приложении используется роутер, реализованный на основе стандартных возможностей браузера (History API). Роутер отвечает за отображение различных представлений (страниц) в зависимости от текущего URL без перезагрузки страницы. Это позволяет реализовать поведение одностраничного приложения (SPA) и улучшить пользовательский опыт.
# Backend
В качестве серверной технологии был выбран Node.js.
Данный выбор обусловлен следующими причинами:
- Использование одного языка (JavaScript) как на клиенте, так и на сервере
- Высокая производительность при работе с асинхронными операциями
- Удобство работы с HTTP-запросами и API
- Широкая экосистема пакетов (npm).
В качестве системы управления базами данных используется MySQL.
Данный выбор обусловлен следующими причинами:
- Реляционная модель данных, удобная для структурированной информации;
- Высокая надёжность и стабильность;
- Поддержка сложных запросов (JOIN, агрегатные функции);
- Широкое распространение и совместимость с Node.js.
# Визуальная часть
- Аутентификация:
    - Блок ввода данных (логин, пароль, выбор роли)
    - Кнопка для входа, регистрации, восстановления пароля и входа в качестве гостя.
- Главная страница:
    - Информация о магазине
    - Кнопки для перехода к каталогу товаров, к профилю/регистрации/входу
    - Поисковая строка в шапке.
- Страница профиля:
    - Данные пользователя (логин/почта/роль)
    - Кнопка выхода из аккаунта
    - Кнопка каталога товаров
    - Кнопка для перехода к корзине.
- Категории товаров:
    - Выбор среди множества различных категорий товаров (с поддержкой подкатегорий)
    - Поисковая строка в шапке
    - Кнопка для перехода к своему профилю/регистрации/входу
    - Кнопка "Назад".
- Страница с товарами:
    - Карточки товаров, найденных по заданной категории или поисковому запросу
    - Кнопка для перехода к своему профилю/регистрации/входу
    - Кнопка для перехода к корзине.
    - Кнопка "Назад".
- Страница товара:
    - Галерея изображений товара
    - Стоимость товара
    - Кнопка добавления товара в корзину
    - Описание товара.
- Корзина:
    - Карточки сохранённых товаров с возможностью изменения количества
    - Итоговая сумма
    - Кнопка "Забронировать" (переносит товары из корзины в бронирования)
    - Кнопка "Продолжить покупки".
- Админ панель:
    - Кнопки для перехода на страницу управления товарами, категориями, администраторами и бронированиями.
- Управление товарами:
    - Таблица товаров с фото, названием, категорией, ценой и наличием
    - Кнопки добавления, редактирования и удаления товара.
- Управление категориями:
    - Таблица категорий с поддержкой иерархии (родительские и дочерние)
    - Кнопки добавления и удаления категории.
- Управление администраторами:
    - Таблица администраторов
    - Кнопки добавления и удаления администратора.
- Бронирования:
    - Таблица забронированных товаров с указанием пользователя
    - Кнопка удаления бронирования.
- Изменение/создание товара:
    - Поля ввода/изменения названия, описания, стоимости, категории
    - Загрузка до 5 изображений товара
    - Переключатель наличия товара
    - Кнопка "Назад".

# Ключевые эндпоинты API

### *Авторизация*
| Метод | Эндпоинт             | Описание                                    |
| ----- | -------------------- | ------------------------------------------- |
| POST  | `/api/login`         | Вход в аккаунт по логину и паролю           |
| POST  | `/api/reg`           | Регистрация нового пользователя             |
| POST  | `/api/recovery`      | Отправка кода восстановления на email       |
| POST  | `/api/reset-password`| Смена пароля по коду из письма              |
| POST  | `/api/logout`        | Выход из аккаунта                           |
| GET   | `/api/check-auth`    | Проверка статуса авторизации                |
| GET   | `/api/profile`       | Получение данных профиля                    |

### *Категории*
| Метод | Эндпоинт              | Описание                                    |
| ----- | --------------------- | ------------------------------------------- |
| GET   | `/api/categories`     | Получение списка категорий                  |
| POST  | `/api/categories`     | Добавление новой категории (админ)          |
| DELETE| `/api/categories/:id` | Удаление категории (админ)                  |

### *Товары*
| Метод | Эндпоинт              | Описание                                    |
| ----- | --------------------- | ------------------------------------------- |
| GET   | `/api/products`       | Получение списка товаров по категории       |
| GET   | `/api/search`         | Поиск товаров по названию/описанию          |
| GET   | `/api/products/:id`   | Получение детальной информации о товаре     |
| POST  | `/api/products`       | Добавление нового товара (админ)            |
| PUT   | `/api/products/:id`   | Редактирование товара (админ)               |
| DELETE| `/api/products/:id`   | Удаление товара (админ)                     |

### *Корзина*
| Метод | Эндпоинт              | Описание                                    |
| ----- | --------------------- | ------------------------------------------- |
| GET   | `/api/cart`           | Получение содержимого корзины               |
| POST  | `/api/cart/add`       | Добавление товара в корзину                 |
| PUT   | `/api/cart/update`    | Изменение количества товара в корзине       |
| DELETE| `/api/cart/:id`       | Удаление товара из корзины                  |

### *Бронирование*
| Метод | Эндпоинт                   | Описание                               |
| ----- | -------------------------- | -------------------------------------- |
| POST  | `/api/reservations/book`   | Бронирование товаров из корзины        |
| GET   | `/api/reservations`        | Получение бронирований пользователя    |
| GET   | `/api/admin/reservations`  | Получение всех бронирований (админ)    |
| DELETE| `/api/admin/reservations/:id`| Удаление бронирования (админ)        |

### *Администраторы*
| Метод | Эндпоинт              | Описание                                    |
| ----- | --------------------- | ------------------------------------------- |
| GET   | `/api/admins`         | Получение списка администраторов (админ)    |
| POST  | `/api/admins`         | Добавление администратора (админ)           |
| DELETE| `/api/admins/:id`     | Удаление администратора (админ)             |
